# Skills obrigatórias neste projeto

Sempre invocar a skill correspondente antes de trabalhar nessas áreas:

| Situação | Skill |
|---|---|
| Criar ou modificar qualquer jogo (`app/games/`) | `futzada-game-patterns` |
| Bug em localStorage, score ou daily reset | `futzada-game-patterns` |
| Adicionar ou editar jogadores em `lib/games/players-db.ts` | `players-db-manager` |
| Criar Zustand store ou migrar estado de localStorage | `zustand-store-ts` |
| Trabalhar com rotas, layouts ou APIs no Next.js | `nextjs-app-router-patterns` |
| Trabalhar com Prisma schema ou queries | `prisma-expert` |
| Refatorar estado de componentes React | `react-state-management` |
| Estilizar com Tailwind CSS (classes, tokens, responsividade) | `tailwind-patterns` |
| Otimizar performance (bundle, Server/Client Components, data fetching) | `nextjs-best-practices` |
| Revisar qualidade de componentes React (waterfalls, memoização) | `react-best-practices` |
| Desenhar novos componentes ou hooks reutilizáveis | `react-patterns` |
| Implementar lógica de jogo (regras, colisão, loop de estado) | `game-development` |
